# Voxel Craft FPS — CLAUDE.md


## プロジェクト概要

ボクセルで構成された世界をファーストパーソン視点で探索・建築するクラフトゲーム。
歩き回るモード（Walk）とビルドモード（Build）を切り替えながら遊ぶ。
マップはローカルファイルに保存・ロードできる。

## 技術スタック

- **レンダリング**: Three.js
- **言語**: TypeScript
- **ビルド**: Vite
- **物理**: Rapier.js (WASM)
- **地形生成**: simplex-noise

最終的には Capacitor 経由で iOS、Electron 経由で Steam に展開予定。
プロトタイプは Web のみ。

## ゲーム仕様

### 視点・カメラ

- ファーストパーソンビュー（FPS視点）固定
- マウス移動でカメラ回転（Pointer Lock API 使用）

### 操作

| 入力 | 歩行モード | ビルドモード |
|------|-----------|-------------|
| W A S D | 移動 | 移動 |
| マウス移動 | カメラ回転 | カメラ回転 |
| Space | ジャンプ | 上昇 |
| Shift | — | 下降 |
| 左クリック | ブロック設置 | ブロック設置 |
| 左長押し | ブロック破壊 | ブロック破壊 |
| スクロール | ブロック種類切替 | ブロック種類切替 |
| Ctrl+S | マップ保存 | マップ保存 |

### ゲームモード

**歩行モード（Walk Mode）**
- 重力・落下あり
- 地形との衝突判定あり
- Space でジャンプ

**ビルドモード（Build Mode）**
- 重力・落下なし（フロート状態）
- 衝突判定なし
- Space で上昇、Shift で下降

モード切替は **画面右上のボタン（Walk / Build）** で行う。
キーボードショートカットは設けない。

### ブロック仕様

初期ブロック種類（5〜8種）：

| ID | 名前 | 備考 |
|----|------|------|
| 1 | 草（Grass） | |
| 2 | 土（Dirt） | |
| 3 | 石（Stone） | |
| 4 | 木（Wood） | |
| 5 | 砂（Sand） | |
| 6 | 水面（Water） | 破壊不可 |

ブロック選択はホットバー UI（画面下部）＋マウススクロールで行う。

### ワールド仕様

- マップサイズ: 固定（プロトタイプ: 64×64×32 ボクセル程度）
- チャンク単位: 16×16×16
- 地形生成: Simplex Noise によるプロシージャル生成（またはフラット地形）
- 保存形式: JSON（.voxmap 拡張子）をローカルファイルに保存・ロード

### 画面 UI 要素

1. **クロスヘア** — 画面中央
2. **モード切替ボタン** — 右上（Walk / Build、アクティブ側をハイライト）
3. **ホットバー** — 画面下部中央（ブロック種類をスロット表示）
4. **保存ヒント** — 右下（「Ctrl+S to save」などのテキスト）

## アーキテクチャ方針

```
src/
├── main.ts              # エントリポイント
├── game/
│   ├── Game.ts          # ゲームループ管理
│   ├── World.ts         # ワールド・チャンク管理
│   ├── Player.ts        # プレイヤー制御・モード管理
│   ├── InputManager.ts  # キーボード・マウス入力
│   └── Raycast.ts       # ブロック選択・レイキャスト
├── voxel/
│   ├── Chunk.ts         # チャンクデータ・メッシュ生成
│   ├── BlockTypes.ts    # ブロック種類定義
│   └── WorldGen.ts      # 地形生成（Simplex Noise）
├── ui/
│   ├── HUD.ts           # HUD 全体管理
│   ├── Hotbar.ts        # ホットバー UI
│   └── ModeButton.ts    # Walk/Build ボタン
└── io/
    └── MapSerializer.ts # JSON 保存・ロード
```

## コーディング規約

- TypeScript strict モードを使用
- クラス設計を基本とする（関数型は補助的に使用）
- Three.js オブジェクトのメモリ管理を意識する（dispose を忘れない）
- チャンクメッシュの再構築は差分更新にとどめる
- コメントは日本語で記述してよい
